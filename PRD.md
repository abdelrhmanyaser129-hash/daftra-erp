## 1. ملخص تنفيذي

### 1.1 تعريف المنتج
**Daftra ERP** هو نظام تخطيط موارد مؤسسات (ERP) متكامل باللغة العربية، مصمم لإدارة المبيعات، المشتريات، المخزون، العملاء، الحسابات، والتقارير للشركات الصغيرة والمتوسطة. يعمل كـ **تطبيق ويب أحادي الصفحة (SPA)** بواجهة مستخدم عربية كاملة مع دعم الاتجاه من اليمين إلى اليسار (RTL).

### 1.2 المشكلة التي يحلها
- تفتقر الشركات الصغيرة والمتوسطة في مصر والوطن العربي إلى نظام ERP ميسور التكلفة باللغة العربية
- الأنظمة الحالية إما باهظة الثمن (SAP, Oracle) أو أجنبية بالكامل ولا تدعم البيئة المحلية (الضريبة المصرية، السجلات التجارية، إلخ)
- صعوبة تتبع الفواتير، المخزون، حركات الخزينة، وأرصدة العملاء في نظام واحد

### 1.3 الحل
نظام ERP متكامل مبني على **Supabase** (Backend-as-a-Service) و **React 19** مع:
- واجهة عربية كاملة (RTL)
- إدارة الفواتير (بيع وشراء)
- إدارة المخزون والمستودعات
- إدارة العملاء والموردين
- الخزينة والحسابات البنكية
- شجرة الحسابات (Chart of Accounts)
- التقارير الموحدة
- صلاحيات متعددة المستويات
- تكامل واتساب لإرسال الفواتير

## 2. رؤية المنتج

### 2.1 الرؤية (Vision)
أن يكون النظام المحاسبي والإداري الأول للشركات الصغيرة والمتوسطة في العالم العربي — مجاني، مفتوح المصدر، سهل الاستخدام، ومتكامل مع البيئة المحلية.

### 2.2 الرسالة (Mission)
تمكين الشركات الصغيرة والمتوسطة من إدارة أعمالها بكفاءة من خلال نظام ERP عربي متكامل يغطي المبيعات، المشتريات، المخزون، الحسابات، والعملاء دون الحاجة إلى خبرة تقنية متقدمة.

### 2.3 القيمة المقترحة (Value Proposition)
- **مجاني تماماً** — لا تكاليف ترخيص أو اشتراك
- **بالعربية** — واجهة ومحتوى بالكامل باللغة العربية
- **متوافق محلياً** — يدعم الضريبة المصرية، السجلات التجارية، الترقيم الضريبي
- **جاهز فوراً** — استضافة على Supabase + Vercel بنقرة واحدة
- **قابل للتوسع** — بنية قابلة للإضافة والتخصيص

### 7.3 تدفق البيانات (Data Flow)

`
User Action -> Component State -> Supabase Query -> PostgreSQL
                                                      |
User <- Component Re-render <- State Update <- Response
`

- كل المكونات تتصل مباشرة بـ Supabase API (لا يوجد Backend Custom)
- إدارة الحالة محلياً في كل مكون بـ useState و useEffect
- المصادقة عبر AuthContext

## 10. سير العمل (Workflows)

### 10.1 إنشاء فاتورة مبيعات كاملة
1. اختيار العميل (بحث بالاسم أو الموبايل -> Autocomplete)
2. إضافة البنود (اختيار منتج، تحديد الكمية، السعر، الخصم، الضريبة)
3. تحديد الخصم العام (نسبة أو قيمة) والتعديل
4. إضافة دفعة مقدمة (اختياري) -> حساب المبلغ المتبقي تلقائياً
5. إضافة معلومات الشحن (اختياري)
6. اختيار المستودع (لتسجيل حركة المخزون)
7. اختيار الخزنة (لتسجيل المعاملة المالية)
8. حفظ:
   - مسودة -> status='draft' (بدون مخزون أو خزينة)
   - حفظ وطباعة -> deposit=0: unpaid / deposit<total: partial / deposit>=total: paid
   - تسجيل حركة المخزون والخزينة والعميل حسب الحالة

### 10.2 حذف فاتورة
1. الضغط على زر حذف -> تأكيد -> استعادة المخزون -> استعادة الخزينة -> استعادة رصيد العميل -> حذف

### 10.3 إرسال فاتورة عبر واتساب
1. جلب رقم الموبايل من clients -> normalizePhone (20+XXXXXXXXX) -> توليد رسالة -> فتح wa.me

### 10.4 تسجيل دفعة عميل
1. اختيار الفاتورة -> إدخال المبلغ -> اختيار وسيلة الدفع -> اختيار الخزنة
2. تحديث حالة الفاتورة (partial/paid) + deposit_amount + remaining_amount
3. تحديث الخزنة + treasury_transactions + رصيد العميل

### 10.5 مرتجع مبيعات
1. اختيار الفاتورة -> تحديد الأصناف والكميات -> اختيار المستودع -> اختيار الخزنة
2. إنشاء returned_invoices + إعادة المخزون + إنقاص الخزنة + تحديث رصيد العميل
